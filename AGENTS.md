# AGENTS.md

## Project overview

Personal website for Matt Obee (mattobee.com), a design technologist specialising in accessibility. Built with Eleventy 3.x, styled with plain CSS processed by LightningCSS, and deployed on Netlify. Blog/notes content is fetched from Sanity CMS at build time.

## Tech stack

- **Static site generator**: Eleventy 3.x (ES Modules)
- **Templating**: Nunjucks (`.njk`) for layouts and pages, Markdown for project content
- **CSS**: Plain CSS with custom properties, processed by LightningCSS. Fluid type and spacing scales generated by Utopia. Light/dark themes via `prefers-color-scheme`.
- **UI components**: Shoelace web components (loaded from CDN)
- **Icons**: Font Awesome (loaded from CDN)
- **Fonts**: Atkinson Hyperlegible (body), Overpass (headings), loaded from Google Fonts
- **CMS**: Sanity (headless). Studio config lives in `mattobeecom/`.
- **Hosting**: Netlify
- **Testing**: Playwright with axe-core for automated accessibility checks
- **Linting/formatting**: ESLint 9 (flat config) + Prettier

## Directory structure

```
src/                    # Eleventy source
  _data/                # Global data (siteData.json)
  _includes/
    layouts/            # Nunjucks layouts (base, default, note, topic)
    header.njk          # Site header partial
    drawer.njk          # Mobile menu drawer partial
    signature.njk       # Author bio partial
    markdownFilter.js   # Nunjucks filter: render Markdown via markdown-it
    randomPhrase.js     # Nunjucks shortcode: random humorous phrase
  images/               # Static images (passthrough copy)
  projects/             # Project pages (Markdown with YAML front matter)
  scripts/              # Client-side JS and Sanity data fetchers
  styles/               # CSS partials, imported into style.css
    themes/             # Light and dark mode custom properties
    utils/              # Utility classes (a11y, spacing, colour, display)
  utils/                # Build-time utilities (Sanity fetcher, logger)
  *.njk                 # Top-level pages (index, about, projects, notes, etc.)
mattobeecom/            # Sanity Studio (separate npm project)
tests/playwright/       # E2E and accessibility tests
dist/                   # Build output (gitignored)
```

## Build and development

```sh
nvm use           # Use Node version from .nvmrc (lts/*)
npm install
npm start         # Dev server on port 8081
npm run build     # Lint + format + build to dist/
```

Environment variables required for Sanity content:

- `SANITY` -- API token
- `SANITY_PROJECT_ID` -- Sanity project ID

## Coding conventions

### Git

- **Commits must be small and atomic.** Each commit should represent a single logical change.
- **Commit messages should be short** -- a concise summary line, no more than 72 characters. Focus on what changed and why, not how.
- Do not bundle unrelated changes in a single commit.

### Templates

- Use Nunjucks (`.njk`) for all layouts, pages, and partials.
- Use Markdown (`.md`) with YAML front matter for project content in `src/projects/`.
- Layouts extend `base.njk`. Pages specify their layout in front matter.

### CSS

- Plain CSS only. No preprocessors, no Tailwind.
- Underscore-prefixed partials (`_base.css`, `_typo.css`) imported into `style.css`.
- Use existing CSS custom properties from `_variables.css` for colours, spacing, and typography.
- Fluid sizing via `clamp()` values -- do not use fixed breakpoints for type or spacing.
- Component styles use BEM-ish naming (`hero__headline`, `nav__item`).
- Utility classes exist in `styles/utils/` for common patterns.

### JavaScript

- ES Modules throughout (`"type": "module"`).
- Follow existing ESLint and Prettier configurations.
- Format with `npm run format` before committing.

## Accessibility

Accessibility is a core concern. The site claims WCAG 2.2 AA conformance.

- Maintain the skip-to-content link in the header.
- Use `aria-current="page"` on active navigation links.
- Provide `aria-label` on icon-only links and buttons.
- Use `role="list"` on styled `<ul>` elements where CSS strips list semantics.
- Use semantic HTML elements appropriately.
- Preserve the `.sr-only` utility class for screen-reader-only content.
- Ensure sufficient colour contrast in both light and dark themes.
- Test changes with `npm run build` and verify no axe-core violations in Playwright tests.

## Content model (Sanity)

Notes use a digital garden maturity model:

- **seedling** -- early, rough idea
- **budding** -- developing, more structured
- **evergreen** -- mature, well-formed
- **mention** -- brief reference
- **journal** -- personal/diary entry

Notes belong to topics. Both are fetched via GROQ queries at build time.

## Testing

Playwright tests run against Netlify deploy previews on PRs to `main`. Tests include:

- Homepage loads correctly
- Site header is visible
- axe-core automated accessibility scan (zero violations expected)

Run locally: `npx playwright test`
